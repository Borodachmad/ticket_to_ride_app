# Ticket to Ride AI

Приложение на Kivy для анализа фотографии поля Ticket to Ride после партии.

Сейчас приложение умеет:
- загрузить фотографию игрового поля;
- выровнять фото относительно эталонной карты;
- ограничить поиск только зонами перегонов;
- сравнить фото с пустой картой и выделить только добавленные объекты;
- попытаться найти вагончики игроков по цвету и форме;
- сгруппировать найденные вагончики в маршруты;
- посчитать очки по базовой шкале Ticket to Ride.

## Как это работает

В основе обработки лежит несколько этапов:

1. Приложение берёт входное фото и уменьшает его до рабочего размера.
2. Фото выравнивается относительно эталонной пустой карты из `assets/map.jpeg`.
3. Из `routes.json` строится маска перегонов, чтобы искать объекты только там, где действительно могут стоять вагончики.
4. Входное фото сравнивается с пустой картой, чтобы убрать напечатанные элементы поля.
5. Для каждого цвета игрока строится HSV-маска.
6. Находятся контуры, затем отбрасываются шум, слишком большие области и фигуры, не похожие на вагончики.
7. Близкие найденные объекты объединяются в маршруты.
8. Для каждого маршрута считается длина и очки.

## Поддерживаемые цвета

Сейчас распознаются цвета:
- красный;
- жёлтый;
- синий;
- зелёный;
- чёрный.

## Подсчёт очков

Используется стандартная шкала Ticket to Ride:

| Длина маршрута | Очки |
|---|---:|
| 1 | 1 |
| 2 | 2 |
| 3 | 4 |
| 4 | 7 |
| 5 | 10 |
| 6 | 15 |

## Структура проекта

- `main.py` — точка входа приложения;
- `app/screens/menu_screen.py` — стартовый экран;
- `app/screens/photo_screen.py` — выбор фотографии;
- `app/screens/results_screen.py` — экран результатов;
- `score_calc.py` — основная логика распознавания и подсчёта;
- `routes.json` — данные по маршрутам карты;
- `compress_routes.py` — утилита для сжатия маршрутов;
- `assets/map.jpeg` — эталонная пустая карта.

## Требования

- Linux;
- Python 3.10+;

## Установка и запуск

### 1. Перейти в проект

```bash
cd /home/boroda/project/1/ticket_to_ride_ai
```

### 2. Создать виртуальное окружение

```bash
python3 -m venv venv
```

### 3. Активировать виртуальное окружение

```bash
source venv/bin/activate
```

### 4. Установить зависимости

```bash
pip install -r requirements.txt
```

### 5. Запустить приложение

```bash
python3 main.py
```

## Как пользоваться приложением

1. Открой приложение.
2. На главном экране нажми `Загрузить фото игрового поля`.
3. Выбери изображение поля после партии.
4. Нажми `Обработать фото`.
5. На экране результатов посмотри найденные цвета, количество маршрутов, количество вагонов и очки.

## Важные замечания

- Эталонная карта сейчас берётся из `assets/map.jpeg`.
- Наилучший результат будет, если фото сделано сверху, без сильных искажений перспективы.
- Если снимок сильно отличается по освещению или ракурсу, точность распознавания снизится.
- Пустая карта должна давать нулевой результат: это базовая проверка корректности текущего алгоритма.
- Алгоритм ещё требует дальнейшей калибровки на реальных фотографиях партий.

## Отладка

Если приложение не запускается:

1. Проверь, что используется `python3`, а не `python`.
2. Проверь, что активировано виртуальное окружение:

```bash
which python3
```

Ожидается путь внутри `venv`.

3. Проверь, что зависимости установлены:

```bash
python3 -c "import cv2, numpy, kivy; print('ok')"
```

4. Если Kivy запускается с предупреждениями про `xinput` или `Xwayland`, это не критическая ошибка и обычно не мешает работе приложения.
